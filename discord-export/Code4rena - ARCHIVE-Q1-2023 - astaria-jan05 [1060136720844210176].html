Can anyone explain the slope/yIntercept concepts in WithdrawProxy and Public Vault?
nogo 01/10/2023 15:10
yIntercept = amount present in the vault?
15:10
slope = the interest rate? (edited)
nogo
yIntercept = amount present in the vault?
santiagogregory 01/10/2023 15:54
not exactly
15:54
basically, the vault has an "implied value" which is what the vault would have in real assets (WETH, for example) if every loan was being linearly repaid
15:55
for example, if the vault has 10 WETH, and a 4 WETH loan thats 50% done, that implied value of the vault is 12 WETH (note that this implied value isnt tracked internally)
15:55
so here your slope would be however many weth/second that loan is accruing
15:56
the publicvault tracks a "last" timestamp which is the last timestamp any loan was repaid, a liquidation happened, or any other state change was made (for example a processEpoch())
15:56
the implied value of the vault is yIntercept + slope * (block.timestamp-last)
15:56
this is how we compute how much LPs should be owed, without having to do an actual state change where we recompute how much interest has been accrued against every active loan
15:56
we essentially move the gas cost to the stage change to whenever a loan repayment or liquidation is made
